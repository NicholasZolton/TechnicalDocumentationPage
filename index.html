What is Java

Java is a truly special language for many people, both because it is widely taught and because it is very powerful. If you want to learn a first programming language, Java is a brilliant choice and I would highly recommend it!

Unlike some other coding languages, Java makes memory management much easier, and you don't have to worry about things like memory leaks or garbage collection.

Java is an Object Oriented Language

This means that Java functions by creating classes and objects. Everything either has a is-a or has-a relationship with something else.

One of the most beautiful things about Object Orientated Languages is the is the ability to model things similarly to how we perceive them in our minds.

For example, let's say we have a car. It has wheels, a seat, an engine, and so much more. We can make each of these objects part of that car. We can then make a car object that has these objects as parts. In doing so, we can then model our perception of the car through code.

Classes and Objects

Classes are like blueprints for objects. Objects are like the actual things that are made from the classes.

We can instantiate objects by calling a class with the "new" keyword and adding a parenthesis after the class name.

When we do this, Java actually allocates memory for the object. In doing so, the object is placed into the heap. The heap is a place where all objects are stored.

On the other hand, many of the variables we see in Java aren't actually objects. Instead, they are simply "primitive variables", or "primitive types". Instead of being stored on the "heap" these primitive variables are stored on the "stack".

Here is an example of some primitive variables.

int x = 5; int y = 10; int z = x + y;

When the function these variables are created in is called, the variables are placed on the stack. When the function ends, the variables are removed from the stack.

Now if we tried to call these variables outside of the function, we would get an error.

{ int x = 5; int y = 10; int z = x + y; }

System.out.println(x); System.out.println(y); System.out.println(z);

This would result in an error.

Objects in Practice

Here is a super basic example of what some code looks like with this syntax.

String name = new String();

Here we called the String constructor from the String class. In doing so, we created a new object. Just like we mentioned earlier, this object was placed on the heap. Because of this, we can use the object we created outside of the function it was originally created in, even if the function has ended.

To destroy this object, we can simply remove all of our references to it. This is done by setting "name" to null. In some other languages we would be able to use the "delete" keyword, but in Java we can't. The delete keyword doesn't actually exist because the Java language uses something called a Garbage Collector.

Here are some other examples of things we would make objects.

  • Dog dog = new Dog();
  • Cat cat = new Cat();
  • Router router = new Router();
  • Person person = new Person();
  • Car car = new Car();

Abstracts

Abstracts are something a little bit unique. While they are a natural extension of the idea of classes, they also break a lot of the rules.

Abstract classes are classes that cannot be instantiated. They are meant to be extended by other classes.

This allows us to create parents of objects that are incomplete in nature, meaning that they don't have the full functionality of an object.

Here's an example of an abstract class.

abstract class Animal { public abstract void makeSound(); }

See how the makeSound() method is abstract and doesn't contain any code? This is because it is not meant to be implemented alone. It is meant to be extended by other classes.

Here's an example of what extending that abstract method looks like.

class Cat extends Animal { public void makeSound() { System.out.println("Meow!"); } }

Now when we call the makeSound() method on a Cat object, we get the output "Meow!".
